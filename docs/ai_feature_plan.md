# AI Feature Plan - 2026-06-14

> 状态：方向已定，暂不开发。等后续有时间再启动。本文记录已敲定的方向、设计决策、待定问题，避免下次从头讨论。

## 1. 定位

不是问答机器人，而是一个「了解用户、帮排解生活困扰的朋友式 AI」。数据源是用户自己写的「笺」/日记（本地 `DayleafDatabase` / `JSONDayleafStore`）。它的价值在于"读懂你今天写的那件具体的事"，而不是给金句或当通用 ChatGPT。

## 2. 数据链与边界（结构图）

详见 `docs/ai_data_chain.svg`。

```
① 碎片记录 ──整理──▶ ② AI 整理日记 ──沉淀──▶ ③ 长期画像 ──陪伴──▶ ④ 朋友式回应
   用户主动写的「笺」      └→ 结构化日记            持续更新的你         读懂你、帮你排解
   （不催/不push）         （富文本/PDF）           （护栏的事实依据）    （当天记录+长期画像）

底座（支撑 ②③④ 全链路）：隐私底座（全本地·自己的key·Keychain） + LLM 调用层（OpenAI 兼容接口）
```

落地顺序：先做确定性最高、风险最低的 **② AI 整理日记**，它顺手产出的干净文本正好是喂给 ③④ 的好语料。

## 3. 已敲定的设计决策

- **API 绑定**：只对接 OpenAI 兼容接口（`base_url` + `api_key` + `model`），覆盖 DeepSeek / Kimi / 通义 / OpenAI / 本地 Ollama。key 进 Keychain，不落明文，不做多供应商适配。配一个「测试连接」按钮。流式输出走 SSE。
- **「不发散」放在事实层，不是整段回答**：情绪共情、通用建议可以发散；凡涉及"用户本人的具体经历"必须有据可查，没有就直说没有、不要编。靠引用具体记录 / 日期做护栏，temperature 调低。
- **朋友感 = 长期记忆**：单独维护一份 AI 持续更新的「长期画像」（压力源、情绪模式、在意的人、目标）。每次回应读「当天记录 + 长期画像」，不是只看今天这一条。这份画像同时是"不发散"的事实依据。
- **不主动打扰（明确否决定时 push）**：产品基调「不打扰、隐私为先」，没有 push、没有"你今天还没记录哦"的催促。AI 只在用户主动动作时在场——触发点用「记录完成」或「用户打开 app」，永远等用户先开口。朋友感来自"我来找它时它懂我"，不是"它追着我跑"。
- **隐私是核心卖点**：全本地存储 + 用户自己的 API key + 不经任何第三方服务器，作为与云端日记 app 的差异点讲清楚。
- **文档形态**：不要让用户碰 Markdown。Mac 原生富文本渲染（AttributedString / SwiftUI），导出给 PDF 或图片，适合存档分享。

## 4. 风险提醒

- **情绪陪伴有边界**：它是朋友不是心理咨询师。用户写到明显的危机情绪时要有兜底（温和建议找人聊、给资源），不能让模型自由发挥去安慰一个真正有风险的人。
- **别变成"正能量复读机"**：只会说"你已经很棒了"的 AI 很快被识破。引用具体细节比给金句重要得多。

## 5. 待定问题（启动开发前要拍）

- **长期画像怎么生成和更新**：每次写完日记增量更新，还是定期重算一次？影响成本和"画像新鲜度"。
- **「笺」的实际字段结构**：开发 ② 之前需要确认一条记录的字段，才能定检索和 prompt 设计。
- **未命中话术**：纯记忆类问题命中不到记录时，怎么引导（例如顺势提示"要不要现在记一笔"），把空答转化成记录行为。
- **中文 embedding 方案**：若 ③ 走语义检索，Apple NLEmbedding 中文支持弱，需走 API embedding 或专门的中文模型。数据量小时可先不上向量库，按时间/关键词召回即可。
